# Implementation Notes - Iteration 6

> **📋 Historical Record**: This document describes implementation details from a specific development iteration. For current architecture and API documentation, see the [Documentation Library](./docs/README.md) and [AI System Guide](./AI_SYSTEM_GUIDE.md).

## Major Changes Implemented

### 1. AI Agent System with Content Moderation

Created a comprehensive multi-agent AI system (`src/lib/ai-agents.ts`) with three specialized agents:

#### Content Moderator Agent
- **Purpose**: Reviews all design prompts for policy violations
- **Age-Based Restrictions**: Automatically blocks NSFW content for users under 18
- **Critical Blocks**: Hate speech, explicit content, violence, illegal activities
- **Response Format**: JSON with violations, severity levels, and alternative suggestions

#### IP Checker Agent
- **Purpose**: Detects potential copyright and trademark violations
- **Detection**: Brand names, characters, celebrity likenesses, copyrighted artwork
- **Context-Aware**: Understands personal use vs commercial context
- **Risk Levels**: none, low, medium, high with specific recommendations

#### Design Assistant Agent
- **Purpose**: Guides users through the design process conversationally
- **Dynamic Responses**: All responses generated by GPT-4 (no hardcoded messages)
- **Sales Script Integration**: Follows structured conversation flow (greeting → discovery → exploration → refinement → generation → approval)
- **Technical Awareness**: Understands DPI, print areas, file formats, and constraints
- **Intent Detection**: Recognizes when users want to generate or approve designs

### 2. Birthday Collection & Age Verification

Updated `AuthDialog.tsx` to collect birthdates:
- **Universal Collection**: Every user must provide birthdate during account creation
- **Age Calculation**: Automatic age calculation from birthdate
- **Minimum Age**: 13 years old to use the service
- **18+ Verification**: Required for NSFW content access
- **Persistent Storage**: Birthdate saved to user profile in database
- **User Type Updates**: Added `birthdate?: string` field to User type

### 3. Product Configuration System

Added support for multiple print location combinations:

#### New Types (`src/lib/types.ts`)
```typescript
interface ProductConfiguration {
  id: string
  name: string          // e.g., "Front Only", "Front & Back"
  printAreas: string[]  // Array of print area IDs
  priceModifier: number // Additional cost for this configuration
}
```

#### Mock Data Updates
Each product now includes configurations:
- **Classic Cotton Tee**: Front Only ($24.99), Back Only ($24.99), Front & Back ($32.99)
- **Long Sleeve Tee**: Front Only ($32.99), Front + Sleeves ($38.99)

This allows users to select specific print combinations with appropriate pricing.

### 4. Google OAuth Configuration in Admin

Created `GoogleOAuthConfig.tsx` component with:
- **Client ID & Secret Management**: Secure storage of Google OAuth credentials
- **Redirect URI Display**: Automatic generation and copy-to-clipboard functionality
- **Setup Guide**: Step-by-step instructions for Google Cloud Console and Supabase
- **Integration Links**: Quick access to Google Cloud Console and Supabase dashboards
- **Checklist**: 8-step setup process clearly documented

Added to Admin Dashboard Settings tab alongside Supabase, Stripe, and Printful configs.

### 5. Fixed Chat System

#### Previous Issue
- Chat would break after first prompt
- No designs were being generated
- Responses were hardcoded instead of using LLM

#### Fixes Applied
- Removed dependency on deprecated `SALES_SCRIPT` module
- Updated all `api.ai` methods to use new agent system
- Made `getInitialMessage()` and `getApprovalMessage()` async
- Added proper error handling for content moderation failures
- Pass `currentUser` to all AI functions for age-based moderation
- All chat responses now dynamically generated by GPT-4

### 6. Content Flow with Moderation

The new flow for design generation:

1. **User sends message** → Design Assistant responds with guidance
2. **User requests generation** (detected by intent keywords)
3. **Content Moderation** runs on the prompt:
   - Checks user age for NSFW restrictions
   - Validates against hate speech, violence, etc.
   - Returns violations if any
4. **IP Check** runs on the prompt:
   - Detects brand names, characters, etc.
   - Returns risk level and recommendations
5. **Design Generation** proceeds if both checks pass
6. **Error Display** shows specific violation messages with suggestions if checks fail

## Known Limitations

1. **Image Generation**: Currently using placeholder SVG generation. Real image generation would require DALL-E or similar API integration.

2. **Age Verification Service**: Using client-side age calculation. Production should use a third-party age verification service for NSFW content.

3. **Configuration Selection**: UI not yet implemented - users currently select products but can't choose specific configurations.

4. **Design Bin**: No visual gallery for managing multiple print area designs yet.

5. **Real-time Updates**: Design files are stored locally in state. Should persist to database for recovery.
